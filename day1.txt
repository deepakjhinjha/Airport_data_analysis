### 2. Understanding Spark Read Modes
Explored different data reading modes to handle real-world messy data:

- **Permissive Mode**
  - Default mode in Spark.
  - Loads all data without failing.
  - Replaces invalid values with nulls.
  - Captures corrupt records in a special column.

- **Drop Malformed Mode**
  - Removes rows that contain invalid or inconsistent data.
  - Useful when bad data is not required.

- **Fail Fast Mode**
  - Immediately stops execution when encountering corrupt data.
  - Suitable for strict data pipelines.
